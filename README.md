# Библиотека коррупционных кейсов — обновленная версия

Рабочий сайт по ТЗ с улучшенным публичным дизайном и уже загруженной стартовой библиотекой кейсов.

Что есть в проекте:

- публичная главная страница с улучшенной визуальной подачей;
- 3 раздела: Россия, иностранные государства, международные организации;
- карточки кейсов и отдельная детальная страница кейса;
- глобальный поиск и поиск внутри раздела;
- фильтры по стране, типу нарушения и году;
- сортировка: сначала новые, по алфавиту, по стране;
- страница «О проекте»;
- закрытая админ-панель с одной ролью `администратор`;
- создание и редактирование кейсов;
- статусы `draft / published / hidden`;
- загрузка фото;
- импорт `.docx` с раскладкой по блокам в режиме best effort;
- справочники: страны, организации, типы нарушений;
- базовая статистика;
- автоматическая генерация PDF для опубликованного кейса;
- стартовое наполнение кейсами из двух подготовленных Word-документов.

## Стек

- Python 3
- стандартный `http.server`
- SQLite
- `python-docx` для Word-импорта
- `reportlab` для PDF
- без внешнего веб-фреймворка

## Как запустить в Ubuntu / WSL

```bash
cd corruption_cases_v2

python3 -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

python app.py
```

Если порт `8080` уже занят, можно запустить на другом:

```bash
CASES_PORT=8091 python app.py
```

После запуска сайт будет доступен по адресу:

```text
http://127.0.0.1:8080
```

или по адресу с тем портом, который ты задал через `CASES_PORT`.

## Данные для входа в админку

По умолчанию:

```text
login: admin
password: admin123
```

Можно переопределить через переменные окружения:

```bash
ADMIN_LOGIN=superadmin ADMIN_PASSWORD=secret123 python app.py
```

## Что важно знать

1. Если у тебя уже есть старая `data.sqlite3`, новый код автоматически дольет стартовые кейсы из `seed_cases.json`, не создавая дубликаты по `slug`.
2. Word-импорт сделан по принципу best effort: файл `.docx` раскладывается по заголовкам и ключевым блокам.
3. PDF доступен только для опубликованных кейсов.
4. Дизайн можно дальше дорабатывать без смены серверной логики.
5. Для продакшена стоит вынести шаблоны, добавить хеширование пароля, CSRF-защиту, более надежные сессии и пагинацию.
